\chapter{Mathematical and Algorithmic Formulations}

\section{Symmetric Authenticated Encryption}
The system exclusively relies on the AES-GCM algorithm for ensuring both plaintext confidentiality and ciphertext integrity. Let $P$ denote the condition-specific patient plaintext (e.g., medical report variables) and let $K_{cond}$ be the 256-bit symmetric key dedicated solely to this medical condition.

The encryption process requires a securely generated $96$-bit initialization vector or Nonce $N$. Furthermore, to prevent key-replay attacks, Additional Authenticated Data (AAD) is constructed using string concatenation of the patient identifier and the document version:
\begin{equation}
AAD = PatientID \parallel ":" \parallel VersionInfo
\end{equation}

The encryption function transforms the inputs into the corresponding Ciphertext $C$ and an Authentication Tag $T$:
\begin{equation}
(C, T) = E_{K_{cond}}(P, N, AAD)
\end{equation}
The final stored artifact in the object store represents the concatenation of the Nonce, Ciphertext, and the Verification Tag. Upon decryption, if $AAD$, $C$, or $N$ have been altered by a single bit, the GCM algorithm mathematically refuses to compute $P$, explicitly triggering a decryption failure \cite{guan2017distributed}.

\section{Shamir's Secret Sharing Scheme}
Once $K_{cond}$ is utilized, storing it poses a critical threat vector. We harness Shamir's Secret Sharing over a finite field to distribute the key material. The underlying concept leverages polynomial interpolation properties.

A polynomial $f(x)$ of degree $k-1$ is constructed, where the constant term represents the secret symmetric key, $f(0) = K_{cond}$. The remaining $k-1$ coefficients $a_1, a_2, \ldots, a_{k-1}$ are chosen completely at random from a finite field $\mathbb{F}_p$ (where $p$ is a large prime number):
\begin{equation}
f(x) = K_{cond} + a_1 x + a_2 x ^ 2 + \cdots + a_{k-1} x ^ {k-1} \pmod p
\end{equation}
Given $n$ available peer nodes, $n$ discrete shares are constructed such that each node $i$ receives the pair $(i, f(i))$. 

When an authorized access request evaluates true, the Trusted Authority aggregates $k$ shares from available and responding nodes. Let the subset of collected shares be represented as $S$. The secret $K_{cond}$ is derived seamlessly using Lagrange Interpolation evaluated at zero:
\begin{equation}
K_{cond} = f(0) = \sum_{j \in S} f(j) \prod_{m \in S, m \neq j} \frac{-m}{j-m} \pmod p
\end{equation}
This mathematical proof guarantees that having exactly $k-1$ shares provides zero determinable hints regarding $K_{cond}$, guaranteeing information-theoretic compartmentalization \cite{li2023privacy}.

\section{Dynamic Threshold Selection Logic}
Different datasets warrant disparate levels of redundancy. The evaluation module maps unstructured English narratives into actionable logic using the classification output from a Large Language Model. Suppose there are a total of $n$ available peers. The priority mapping is evaluated against maximum systemic integrity goals:
\begin{itemize}
    \item \textbf{HIGH Priority:} (e.g., immediate cardiac evaluation data). The data needs rapid turnaround and must survive multiple node failures. In the implemented policy (for $N=5$ peers), this corresponds to a lower threshold $k=2$, enabling reconstruction even when multiple peers are unavailable.
    \item \textbf{MEDIUM Priority:} The parameter is balanced. In the implemented policy (for $N=5$ peers), this corresponds to $k=3$.
    \item \textbf{LOW Priority:} Data that is non-critical and typically requires stronger compromise resistance. In the implemented policy (for $N=5$ peers), this corresponds to $k=4$, enforcing stricter multi-peer cooperation at the expense of fault tolerance \cite{bhandari2016priority}.
\end{itemize}

\section{Node Key Wrapping Protocol}
For a peer identifier $pid$, let the internal Node Master Key be $NMK_{pid}$. When Shamir evaluates to $Share_j$, it cannot be transmitted or logged onto the blockchain openly. The Trusted Authority performs a wrapping layer essentially operating as:
\begin{equation}
WrappedShare_j = AES\_GCM\_Encrypt_{NMK_{pid}}(Share_j, Nonce', AAD)
\end{equation}
The specific inclusion of AAD guarantees that the share strictly binds to the document context. During reconstruction, the respective peer node decrypts the $WrappedShare_j$ strictly using its own restricted $NMK_{pid}$ space, providing the $Share_j$ back to the TA exclusively over TLS-protected intra-network boundaries. This mechanism establishes impenetrable forward and backward security structures against blockchain-level observers.
