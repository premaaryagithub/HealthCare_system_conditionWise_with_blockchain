\begin{center}
{\Large{\bf{ABSTRACT}}
\addcontentsline{toc}{chapter}{Abstract}
}
\end{center}


The main aim of the project is to address the severe security and privacy vulnerabilities inherent in contemporary centralized healthcare databases. By proposing a decentralized ecosystem leveraging Hyperledger Fabric, the system securely manages Electronic Health Records (EHRs) using an immutable cryptographic audit ledger without subjecting massive data payloads to catastrophic single point of failures. Crucially, the system introduces a "condition-wise" access philosophy employing AES-GCM methodologies. A given patient's records are segregated strictly by medical conditions. The master key material is aggressively compartmentalized utilizing Shamir's Secret Sharing mechanisms, distributing threshold shares amongst disparate consensus nodes securely bounded by explicit Node Master Keys. Furthermore, the architecture seamlessly accommodates dynamic fault-tolerance limits explicitly parsed by a Large Language Model evaluating individual text reports for medical urgencies. This structure categorically eliminates the dependencies generally forced upon single absolute Trusted Authorities and explicitly refutes Orderer-as-TA assumptions. Overall, the evaluated framework displays phenomenal latency efficiencies under emergency boundaries alongside uncompromising compromise-resistance capabilities.
